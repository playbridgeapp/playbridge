## 2024-05-24 - [Compose LazyColumn Performance] **Learning:** Using `forEach` inside a single `item` block of a `LazyColumn` defeats the purpose of list virtualization because all items are still composed simultaneously. To properly gain performance benefits, large lists must be unwrapped and mapped using the `items(...)` DSL native to `LazyColumn`. **Action:** When refactoring a `Column` loop into a `LazyColumn`, always ensure the list is mapped using `items(list)` so that Compose can lazily instantiate the individual UI components as they scroll onto the screen.
